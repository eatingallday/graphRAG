  分类别 FN 原因分析

  1. Crypto（4 FN / 5 total，Recall 20%）

  APK: BlockCipher-ECB
  漏洞本质: AES/ECB 模式 → 信息泄露
  管线失败原因: API误用：Cipher.getInstance("AES") 默认
    ECB。不是数据流问题，管线无 crypto API 审计能力
  ────────────────────────────────────────
  APK: BlockCipher-NonRandomIV
  漏洞本质: AES/CBC + 非随机 IV
  管线失败原因: API误用：IV 是否随机需要语义理解，不是 source→sink 模式
  ────────────────────────────────────────
  APK: ConstantKey
  漏洞本质: 密钥硬编码在源码中
  管线失败原因: 密钥硬编码：需扫描 SecretKeySpec 的参数是否来自常量。SAST
    工具检测到了但 validation_agent 缺乏独立判定路径
  ────────────────────────────────────────
  APK: ExposedCredentials
  漏洞本质: Keystore setEntry(null)
  管线失败原因: API误用：ProtectionParameter=null 是参数值问题

  类别结论：Crypto 漏洞 100% 是 API 
  误用/配置问题，不涉及数据流。管线需要一条独立于 taint path 的"Crypto API
  审计"判定路径。

  ---
  2. ICC（13 FN / 17 total，Recall 24%）

  失败模式: Manifest 属性漏洞
  APK 数量: 5
  涉及 APK: TaskAffinity×4, UnhandledException
  根因: 漏洞在 taskAffinity/allowTaskReparenting 属性配置。manifest_agent
    没有检查这些属性
  ────────────────────────────────────────
  失败模式: 动态注册/隐式 Intent
  APK 数量: 4
  涉及 APK: DynamicRegBroadcast, HighPriority, ImplicitPendingIntent,
    StickyBroadcast
  根因: 漏洞在代码中的
    registerReceiver/PendingIntent/sendStickyBroadcast，manifest
    无声明，icc_bridge 检测到模式但 validation_agent 无法从图中构建攻击证据
  ────────────────────────────────────────
  失败模式: 缺乏输入验证
  APK 数量: 3
  涉及 APK: FragmentInjection, IncorrectHandlingImplicitIntent, NoValidityCheck
  根因: 需要理解"intent 参数未验证"的语义，是逻辑漏洞
  ────────────────────────────────────────
  失败模式: PendingIntent 空 Intent
  APK 数量: 1
  涉及 APK: EmptyPendingIntent
  根因: PendingIntent.getActivity(new Intent()) 是 API 误用模式

  类别结论：
  - 5个 TaskAffinity/Exception → manifest 属性审计缺失（需检查 taskAffinity,
  allowTaskReparenting, exported 等组合）
  - 4个动态 ICC → icc_bridge 检测到了模式但 validation_agent 无法利用
  - 4个语义/逻辑漏洞 → 需要代码语义理解，超出当前规则匹配能力

  最高 ROI 的三个优化方向：

  1. 增加 "SAST-driven 独立判定"路径（解决 ~30 个 API 误用 FN）—
  validation_agent 对 SAST 高分 findings 直接判定，不依赖 taint path
  2. 扩展 manifest_agent 属性检查（解决 5 个 ICC FN）— 检查 taskAffinity,
  allowTaskReparenting 等
  3. 扩展 taint source 规则库（解决 ~5 个 Storage/ICC FN）— 加入 ContentProvider
   参数、Intent extras
